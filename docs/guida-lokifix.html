1 Cos'e LokiFix?

LokiFix e un sistema che ti permette di controllare un PC Windows remoto usando Claude Code (l'assistente AI di Anthropic), senza dover installare Claude Code sulla macchina remota.

In parole semplici: tu parli con Claude Code sul tuo PC, e lui puo eseguire comandi, leggere file, controllare servizi e molto altro sul PC del cliente — tutto attraverso un tunnel sicuro.

ℹ️
Caso d'uso tipico: Un tecnico IT deve diagnosticare un problema su un PC remoto. Invece di installare software pesante o chiedere al cliente di seguire istruzioni complicate, gli manda un singolo file .exe. Il cliente lo avvia, condivide un codice di 6 caratteri, e il tecnico puo operare sulla macchina tramite Claude Code.

Vantaggi rispetto ad altre soluzioni

  • Zero installazione sul remoto — un singolo .exe portabile, nessun installer
  • Nessuna porta da aprire — il tunnel e in uscita, funziona anche dietro firewall/NAT
  • Assistenza AI integrata — Claude Code analizza e suggerisce soluzioni in tempo reale
  • Tracciabilita completa — ogni azione viene registrata in un audit log a prova di manomissione
  • Protezione operazioni pericolose — il PC remoto chiede conferma prima di azioni rischiose

2 Come Funziona (Architettura)

LokiFix e composto da due programmi che comunicano tra loro attraverso un tunnel Cloudflare:

   PC OPERATORE (il tuo PC)                        PC REMOTO (il PC del cliente)
  +---------------------------+                   +---------------------------+
  |                           |                   |                           |
  |   Claude Code (CLI/IDE)   |                   |   lokifix-agent.exe       |
  |         |                 |                   |         |                 |
  |    (stdio JSON-RPC)       |                   |    (esegue comandi,       |
  |         |                 |                   |     legge file, ecc.)     |
  |         v                 |                   |         ^                 |
  |   lokifix-mcp.exe         |                   |         |                 |
  |    +------------------+   |                   |   +------------------+    |
  |    | WebSocket Server |   |                   |   | WebSocket Client |    |
  |    | Auth Manager     |   |                   |   | Audit Logger     |    |
  |    | MCP Protocol     |   |                   |   | Danger Detector  |    |
  |    +--------+---------+   |                   |   +--------+---------+    |
  |             |             |                   |            |              |
  +-------------|-------------+                   +------------|-------------+
                |                                              |
                v                                              |
        +---------------+          INTERNET            +-------+-------+
        |  cloudflared  | <-- tunnel TLS criptato --> |  connessione  |
        |  (tunnel)     |    (*.trycloudflare.com)     |  in uscita    |
        +---------------+                              +---------------+

Flusso dei dati passo-passo

1
Tu scrivi a Claude Code Esempio: "Controlla i servizi in esecuzione sul PC remoto"
2
Claude Code chiama il tool MCP remote_services La richiesta arriva a lokifix-mcp.exe tramite il protocollo stdio (JSON-RPC)
3
lokifix-mcp.exe impacchetta la richiesta Crea un "envelope" (busta) con ID univoco, tipo "request", timestamp, e la invia via WebSocket
4
La richiesta viaggia attraverso il tunnel Cloudflare Criptata TLS, passa per i server Cloudflare e arriva al PC remoto
5
lokifix-agent.exe riceve la richiesta L'agente esegue il comando (in questo caso, Get-Service in PowerShell)
6
Il risultato torna indietro Stesso percorso al contrario: agent → tunnel → mcp → Claude Code
7
Claude Code ti mostra il risultato E puo anche analizzarlo, trovare anomalie, e suggerirti azioni correttive

3 I Due Componenti

lokifix-mcp.exe Lato Operatore

Il "ponte" tra Claude Code e il PC remoto. Si avvia automaticamente quando Claude Code ha bisogno dei tool remoti.

  • Comunica con Claude Code via stdio (standard input/output)
  • Gestisce l'autenticazione (codici e token)
  • Avvia il tunnel Cloudflare
  • Traduce le richieste MCP in messaggi WebSocket
  • Genera il codice di connessione da condividere

lokifix-agent.exe Lato Remoto

L'esecutore sul PC del cliente. Un singolo file .exe portabile, nessuna dipendenza.

  • Si connette al tunnel in uscita (nessuna porta da aprire)
  • Esegue comandi PowerShell/CMD
  • Gestisce operazioni su file
  • Raccoglie info di sistema
  • Chiede conferma per operazioni pericolose
  • Scrive audit log di ogni azione

cloudflared.exe Componente di rete

cloudflared e uno strumento ufficiale di Cloudflare che crea un tunnel sicuro tra i due PC. Non serve avere un account Cloudflare — il servizio "Quick Tunnels" e gratuito e assegna un URL temporaneo tipo abc123.trycloudflare.com.

✅
Perche Cloudflare? Perche il PC remoto non deve aprire porte o configurare il firewall. La connessione e sempre in uscita, esattamente come quando navighi su un sito web. Funziona anche dietro NAT, proxy aziendali, e reti ristrette.

Setup automatico (consigliato)

Apri PowerShell nella cartella del progetto ed esegui:

.\setup.ps1

Lo script fa tutto automaticamente:

  1. Verifica che Go sia installato
  2. Compila lokifix-agent.exe e lokifix-mcp.exe nella cartella build/
  3. Scarica cloudflared.exe da GitHub (se non gia presente)
  4. Configura Claude Code registrando il server MCP
  5. Imposta i permessi per i 16 tool remoti

Setup manuale (se necessario)

Passo 1 — Compila:

go build -ldflags="-s -w" -o build/lokifix-agent.exe ./cmd/lokifix-agent/
go build -ldflags="-s -w" -o build/lokifix-mcp.exe ./cmd/lokifix-mcp/

Passo 2 — Registra in Claude Code CLI:

claude mcp add lokifix-remote -- "C:\percorso\build\lokifix-mcp.exe"
⚠️
Attenzione: Per Claude Code CLI usa il comando claude mcp add. Il file claude_desktop_config.json funziona solo per Claude Desktop (app grafica), non per la CLI.

Step-by-step dettagliato

Sul tuo PC (operatore):

  1. Apri Claude Code (CLI o IDE)
  2. Usa un qualsiasi tool remoto, ad esempio chiedi a Claude: "Mostrami le info di sistema del PC remoto"
  3. Claude Code avvia automaticamente lokifix-mcp.exe
  4. Il server genera un codice di connessione di 6 caratteri (es. AB3XK7)
  5. Il codice viene anche salvato in ~/lokifix-connection.txt
  6. Comunica questo codice al cliente (telefono, chat, email)

Sul PC remoto (cliente):

  1. Copia lokifix-agent.exe sul PC (no installazione necessaria)
  2. Fai doppio clic per avviarlo
  3. Inserisci il codice di connessione quando richiesto
  4. L'agente si connette — vedrai il messaggio di conferma
ℹ️
Cosa vede il cliente: Una finestra di terminale con un banner LokiFix, un prompt per inserire il codice, e poi un log in tempo reale di tutte le operazioni eseguite. Il cliente puo vedere tutto quello che fai.

Il codice di connessione nel dettaglio

Il codice di connessione (es. AB3XK7) e in realta un contenitore codificato che include:

  • L'URL del tunnel Cloudflare (dove connettersi)
  • Un token di autenticazione di 32 byte (la "password")

Il formato interno e: URL_websocket|token_base64, il tutto codificato in base64-url.

Il codice di connessione scade dopo 15 minuti. Se non viene usato entro quel tempo, ne va generato uno nuovo.

⚠️
Alfabeto speciale: I codici usano solo ABCDEFGHJKLMNPQRSTUVWXYZ23456789 (32 caratteri). Sono esclusi 0, O, 1, I per evitare confusione quando li detti al telefono.

6 Sistema di Autenticazione

LokiFix usa un sistema a due livelli per l'autenticazione:

Livello 1: Codice di Connessione Monouso

  • Generato all'avvio del server MCP
  • 6 caratteri alfanumerici
  • Scade dopo 15 minuti
  • Si usa una sola volta
  • Contiene URL del tunnel + token

Livello 2: Session Token Persistente

  • Generato dopo la prima autenticazione
  • Token di 32 byte random
  • Dura 24 ore
  • Permette la riconnessione automatica
  • Nessun bisogno di reinserire il codice

Flusso di autenticazione

PRIMA CONNESSIONE:                        RICONNESSIONE:

Agent invia:                              Agent invia:
  { token: "abc123..." }                    { sessionToken: "xyz789..." }
         |                                          |
         v                                          v
  Server verifica il token                 Server verifica il sessionToken
  del codice di connessione                (confronto costante nel tempo)
         |                                          |
         v                                          v
  OK! Genera sessionToken                  OK! Connessione ripristinata
  e lo invia all'agent                     (nessun nuovo token generato)
         |
         v
  Agent salva il sessionToken
  per le riconnessioni future
✅
Riconnessione automatica: Se la connessione cade (rete instabile, PC in sleep, ecc.), l'agente riprova automaticamente fino a 5 volte con attesa crescente (2s, 4s, 6s, 8s, 10s). Usa il session token, quindi non serve reinserire il codice.

Sicurezza dell'autenticazione

  • Confronto a tempo costante — Il server confronta i token con subtle.ConstantTimeCompare, che previene attacchi di timing (non si puo indovinare il token misurando il tempo di risposta)
  • Token generati con crypto/rand — I token usano il generatore di numeri casuali crittograficamente sicuro del sistema operativo
  • Nessuna password memorizzata — I token esistono solo in memoria, mai scritti su disco in chiaro
  • Timeout di 5 secondi — Se l'agente non si autentica entro 5 secondi dalla connessione WebSocket, viene disconnesso

8 Sicurezza e Protezioni

LokiFix implementa molteplici livelli di sicurezza per proteggere il PC remoto:

Rilevamento Operazioni Pericolose

L'agente analizza ogni richiesta prima di eseguirla. Se rileva un'operazione potenzialmente pericolosa, chiede conferma all'utente sul PC remoto:

+======================================================+
|  !! OPERAZIONE PERICOLOSA - CONFERMA RICHIESTA       |
+======================================================+
|  Azione:    SHELL_EXEC                               |
|  Dettaglio: Remove-Item C:\Users\mario\Desktop\*     |
|  Motivo:    Pattern 'Remove-Item' rilevato           |
+======================================================+
                  Approvi? [s/N]: _

Comandi shell considerati pericolosi:

CategoriaPattern rilevati
Eliminazionerm, del, rmdir, Remove-Item
Formattazione discoformat, diskpart, cipher /w
Controllo serviziStop-Service, Stop-Process, Restart-
Registroreg delete, reg add, Set-ItemProperty
Sistemashutdown, taskkill, bcdedit, sfc, dism
ReteDisable-NetAdapter

Path di file considerati sensibili:

CategoriaPath protette
Sistema WindowsC:\Windows\, \System32\, \SysWOW64\
ProgrammiC:\Program Files\, C:\ProgramData\
File critici.exe, .dll, .sys, .bat, .ps1, .reg, hosts

Crittografia

  • Tutto il traffico passa attraverso un tunnel TLS di Cloudflare
  • La connessione e criptata end-to-end
  • Nessun certificato da gestire manualmente
  • L'URL del tunnel e casuale e cambia ad ogni sessione

Superficie di attacco minima

  • Nessuna porta aperta sul PC remoto — la connessione e solo in uscita
  • Nessuna credenziale salvata — solo token in memoria
  • Codice monouso — valido solo 15 minuti
  • 20 richieste contemporanee max — semaforo che limita l'esecuzione parallela
🚨
Regola d'oro: Il PC remoto ha sempre l'ultima parola. L'utente sul PC remoto puo rifiutare qualsiasi operazione pericolosa, e puo chiudere l'agente in qualsiasi momento con Ctrl+C.

9 Audit Log e Tracciabilita

Ogni azione eseguita da LokiFix viene registrata in un audit log a prova di manomissione.

Come funziona l'audit log

Ogni voce del log include un hash SHA-256 che incorpora l'hash della voce precedente, creando una catena (come una mini-blockchain). Se qualcuno modifica una voce, la catena si spezza e la manomissione e rilevabile.

Voce 1                  Voce 2                  Voce 3
+------------------+    +------------------+    +------------------+
| hash: abc123     |    | hash: def456     |    | hash: ghi789     |
| prev: GENESIS    |--->| prev: abc123     |--->| prev: def456     |
| azione: SYSINFO  |    | azione: SHELL    |    | azione: FILE_READ|
| risultato: OK    |    | risultato: OK    |    | risultato: OK    |
+------------------+    +------------------+    +------------------+

Dove si trovano i log

PCPosizioneNome file
Operatore~/lokifix-logs/lokifix-OPERATOR-{session}-{timestamp}.log
RemotoVicino a lokifix-agent.exe in lokifix-logs/lokifix-REMOTE-{session}-{timestamp}.log

Cosa viene registrato

  • Ogni esecuzione di tool (comando, file letto/scritto, info richieste)
  • Le conferme dell'utente (APPROVED / DENIED) per operazioni pericolose
  • Inizio e fine sessione
  • Errori di connessione

Report di sessione

Quando l'agente si disconnette, viene generato automaticamente un report HTML con il riepilogo della sessione: quante operazioni eseguite, di che tipo, eventuali errori.

10 Troubleshooting

Problemi comuni e soluzioni

ProblemaCausa probabileSoluzione
L'agente non si connette Codice scaduto (>15 min) Riavvia lokifix-mcp per generare un nuovo codice
Errore 502 dal tunnel Istanze precedenti di cloudflared ancora attive Chiudi tutti i processi cloudflared e lokifix-mcp, poi riavvia
Claude Code non trova i tool remoti MCP non registrato correttamente Esegui claude mcp add lokifix-remote -- "percorso\lokifix-mcp.exe"
L'agente si disconnette spesso Rete instabile L'agente riprova automaticamente (5 tentativi). Se persiste, verifica la connessione Internet del PC remoto
Comandi lenti sul remoto Timeout di default troppo breve I comandi hanno timeout di 120s. Per comandi piu lunghi, specificare un timeout maggiore
Errore "operazione pericolosa" non voluto Pattern di sicurezza troppo ampio L'utente sul PC remoto deve semplicemente approvare con s. Il pattern e conservativo per sicurezza
Setup fallisce su "Go not found" Go non installato o non nel PATH Installa Go da go.dev e riavvia il terminale
